How it works

Spend the budget on coding, not context bloat.

Path kickoff

Start with token-reduce-paths.sh so the agent gets candidate files without paying for a fat snippet payload.

Guardrails

Bash, Glob, and Grep hooks block exploratory scans and push the workflow back onto the cheapest discovery path.

Architecture

The value is the whole system: guidance, helper scripts, hooks, and install surfaces. See references/architecture.md for the high-level flow.

Use this when

  • you do not know where the answer lives yet
  • the repo is large enough that `find .` and `rg --files .` are noisy
  • you want the same path-first behavior in Claude, Codex, and MCP clients

Skip this when

  • the exact file path is already known
  • the task is a small single-file edit
  • you are trying to change Codex or Claude internals

Boundary

This does not change Codex behavior and it does not add caching to Claude Code's own built-in session handling. It only helps when you own the Anthropic request payload yourself.

FAQ

What tends to confuse people on first read.

Does this always save tokens?

No. Path-first discovery is the cheap default. Ranked snippet mode is a follow-up path and can cost more on a tiny repo.

Does this require QMD?

No. QMD is preferred when available, but scoped `rg` remains the fallback.

Does the Anthropic helper affect Codex or Claude Code sessions?

No. It only helps when you own the Anthropic API payload and can place `cache_control` markers yourself.
